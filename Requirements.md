## 1. 프로젝트 개요

### 1.1 목적

부동산 경매 물건에 대해 아래 항목을 자동화하여 **단기 매매사업자의 의사결정을 지원**하는 시스템 구축

- 데이터 수집
- 권리 분석
- 시세 비교
- 수익성 분석
- 입찰가 추천

### 1.2 핵심 가치

| 항목 | 개선 내용 |
|------|-----------|
| 분석 시간 단축 | 1시간 → 5분 |
| 권리 리스크 식별 | 자동화 |
| 입찰가 산정 | 자동 산출 |
| 투자 추천 | 개인화 |

### 1.3 타겟 사용자

- 단기 매매사업자
- 경매 투자자
- 부동산 중개/투자 전문가

## 2. 시스템 범위 (MVP)

### 2.1 대상 범위

| 구분 | 내용 |
|------|------|
| 물건 유형 | 아파트, 오피스텔, 단독주택, 상가, 꼬마빌딩 |
| 지역 | 초기특정지역 : 부산 |
| 데이터 | 공개 경매 데이터 + 실거래가 |

### 2.2 MVP 목표

> **"경매 물건 분석 자동화"**

---

## 3. 기능 요구사항

### 3.1 경매 데이터 수집

**기능**
- 경매 물건 목록 크롤링
- 사건 단위 데이터 저장

**수집 데이터**

| 필드 | 설명 |
|------|------|
| 사건번호 | 법원 사건 식별자 |
| 법원 | 관할 법원명 |
| 물건번호 | 물건 식별자 |
| 주소 | 소재지 |
| 물건종류 | 아파트 / 오피스텔 등 |
| 감정가 | 감정평가 금액 |
| 최저가 | 입찰 최저 금액 |
| 유찰 횟수 | 누적 유찰 수 |
| 입찰 기일 | 입찰 예정일 |
| 면적 | 전용/공급 면적 |
| 층수 | 해당 층 |
| 이미지 URL | 물건 사진 링크 |

**요구사항**
- 크롤링 실패 시 retry
- 데이터 중복 방지
- 변경 감지 (유찰, 가격 변경)

---

### 3.2 문서 수집 및 파싱

**대상 문서**
- 등기부등본
- 매각물건명세서
- 현황조사서
- 감정평가서 (가능 시)

**기능**
- PDF 다운로드
- 텍스트 추출
- 구조화된 데이터 변환

**출력 구조 예시**

```json
{
  "ownership": [],
  "mortgage": [],
  "tenant": [],
  "risk_flags": []
}
```

---

### 3.3 시세 데이터 수집

**데이터**
- 실거래가
- 매물가
- 동일 평형 비교
- 최근 거래 트렌드

**기능**
- 주소 기반 매칭
- 유사 물건 필터링
- 평균 / 중앙값 계산

---

### 3.4 권리 분석 (Rule Engine)

**분석 항목**
- 말소기준권리
- 인수 권리 여부
- 임차인 대항력
- 우선변제권
- 가압류 / 가처분
- 법정지상권 가능성

---

### 3.5 수익성 분석

**입력**

| 항목 | 설명 |
|------|------|
| 예상 매도가 | 시세 기반 추정 매도 금액 |
| 수리비 | 예상 수선 비용 |
| 명도비 | 명도 처리 비용 |
| 세금 | 취득세, 양도세 등 |
| 금융비용 | 대출 이자 등 |

**계산식**

```
최대 입찰가 = 예상 매도가 - 총비용 - 목표이익
```

**출력**
- 보수적 입찰가
- 적정 입찰가
- 공격적 입찰가

---

### 3.6 점수화 시스템

**점수 항목**
- 권리 안전성
- 수익성
- 입지
- 명도 난이도
- 경쟁 강도

---

### 3.7 AI 분석 (LLM)

**역할**
- 문서 요약
- 리스크 설명
- 투자 코멘트 생성

**입력**
- 구조화 데이터
- 원문 텍스트

**클루드코드 API KEY**
[API KEY - 보안을 위해 제거됨. 환경변수 ANTHROPIC_API_KEY 사용]

---

### 3.8 추천 시스템

**기능**
- 사용자 조건 기반 필터링
- 점수 기반 정렬
- 추천 리스트 생성

**사용자 설정**
- 투자 목적
- 예산
- 지역
- 목표 수익률
- 리스크 허용도

---

### 3.9 포트폴리오 관리

**기능**
- 관심 물건 저장
- 상태 추적 (유찰 / 낙찰)
- 결과 기록

## 4. 데이터 모델 (초안)

**주요 엔티티**

| 엔티티 | 설명 |
|--------|------|
| `AuctionCase` | 경매 사건 |
| `AuctionProperty` | 경매 물건 |
| `AuctionSchedule` | 입찰 일정 |
| `RegistryDocument` | 등기부 문서 |
| `OccupancyInfo` | 점유 정보 |
| `MarketPrice` | 시세 데이터 |
| `RiskFlag` | 리스크 플래그 |
| `Recommendation` | 추천 결과 |
| `BidStrategy` | 입찰 전략 |
| `UserPreference` | 사용자 설정 |
| `Watchlist` | 관심 목록 |
| `OutcomeHistory` | 결과 이력 |

## 6. 시스템 아키텍처

| 구성 요소 | 기술 스택 |

## 8. 리스크 및 정책

### 8.1 리스크

- 크롤링 차단
- 데이터 불일치
- 주소 매칭 오류

### 8.2 정책

- 문서 누락 시: 분석 보류
- 리스크 발견 시: 경고 플래그
- LLM은 설명만 수행 (판단은 룰엔진 담당)

---

## 9. 성공 기준

- 분석 시간 80% 단축
- 추천 정확도 향상
- 사용자 재사용률 증가
